Architecture

Governed dataflow from chat to execution

Lisa keeps routes thin. Real decisions move through LisaCore, constitutional mode selection, brain orchestration, model routing, capability checks, sandbox runtime, audit logging, and response rendering.

  1. 01 Chat webhook receives Telegram, Slack, or WhatsApp message
  2. 02 MessageNormalizer and CommandParser produce internal command shape
  3. 03 Judicial Police selects Normal, Autopilot, or Spike Constitution
  4. 04 Brain swarm plans, scores, critiques, and requests approvals
  5. 05 TokenBankBroker and ModelRouter select the best available model
  6. 06 SandboxRuntime isolates risky execution and records results
  7. 07 AuditLog captures every meaningful decision and response

Modes

Normal, Autopilot, and Spike run under separate constitutions

Normal Constitution

Strict, plan-first behavior for regular chat and /lisa plan. Low token usage and no autonomous execution by default.

Autopilot Constitution

Expanded autonomy for explicit Autopilot commands: brain swarm planning, sandboxed execution, token-bank routing, and approval gates.

Spike Constitution

Exploratory mode for research and prototypes. Allows comparison and sandbox experiments, but no irreversible changes.

Safety model

Powerful does not mean lawless

Autopilot may relax normal bottlenecks, but it never bypasses the Judicial Police, audit logs, approval gates, sandbox isolation, Principal approval, or tool quarantine.

Always blocked

  • Disabling Judicial Police or AuditLog
  • Bypassing SandboxRuntime
  • Secret access without approval
  • Merge, deploy, or database writes without approval
  • Destructive shell patterns like rm -rf, curl | sh, chmod 777

Approval required

  • Package, MCP, CLI, or tool activation
  • Large token-bank reservations
  • Sandbox network enablement
  • Policy or constitution changes
  • External messages outside the active user channel
